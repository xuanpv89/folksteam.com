CMS offline-ready: trang này không dùng CDN nên sẽ mở được ngay. Nó tạo file Markdown đúng schema blog hiện tại. Sau khi tải file, đặt vào thư mục tương ứng rồi commit/deploy.

Sau khi điền form thì làm gì?

  1. 1
    Bấm Download .md để tải file bài viết, hoặc Copy Markdown nếu muốn dán thủ công.
  2. 2
    Đặt file vào đúng đường dẫn đang hiển thị bên dưới form, ví dụ src/content/blog/en/test-1.md.
  3. 3
    Hoặc dùng mục Publish to GitHub trong form để commit bài trực tiếp lên repo. Khi deploy build xong, bài sẽ tự hiện trên trang Blog và filter theo category.

Publish to GitHub

Dùng GitHub fine-grained token có quyền Contents: Read and write cho repo này. Token chỉ được lưu trong trình duyệt của bạn.

Sau khi bấm Publish to GitHub, CMS sẽ tạo commit file Markdown vào repo. Vercel sẽ tự deploy lại nếu project đang kết nối với GitHub.
